## 2026-05-14 - Eliminate redundant ActiveTeam query in RenderTemplate\n**Learning:** When retrieving the active team's name for layout rendering, the original code performed an N+1 style database query despite having just fetched the user's teams in the exact same function. Extracting the `teams` slice earlier in the scope and iterating over it reduces DB load by 1 query per page render for authenticated users.\n**Action:** Always check if required template data has already been fetched in the current request context before defaulting to another database query, and ensure variables are typed properly (e.g., `[]map[string]interface{}`) when reusing them.
